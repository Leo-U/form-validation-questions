Study Questions

A1: Form validations are specific constraints or rules that determine what data users can enter into an input. When a user enters data that breaks the rules, a message will appear, providing feedback on what was wrong with the entered data and how to fix it.

A2: You can validate the minimum and maximum of numeric inputs using the min and max attributes. These attributes allow you to set the lower and upper bounds of the value entered into the form control.

A3: The pattern validation can be used to ensure data matches a particular pattern. Real-world applications would be checking if a credit card number or a zipcode is in the correct format.

A4: The required validation ensures that specific fields have been filled in before submitting the form. For example, the email and password in a login form.

A5: You can use the minlength and maxlength attributes to validate the length of text inputs. The minlength attribute sets the minimum number of characters, while the maxlength attribute sets the maximum number of characters.

A6: The :valid and :invalid pseudo CSS selectors are available for styling valid and invalid inputs respectively.

A7: Built-in validations have limitations such as not being able to validate that a password input and password confirmation input have the same value or validating that a username has not already been taken. They also limit what can be done with styling the validation messages and the content within them.

A8: To indicate which fields are required in a form, you can add an asterisk (*) to the required field label. This is often done to ensure a good user experience and to meet accessibility guidelines.

A9: To add a minimum length validation to a form control, you can use the minlength attribute with an integer value that represents the minimum number of characters you want to allow in the form control.

A10: To add a maximum length validation to a form control, you can use the maxlength attribute with an integer value that represents the maximum number of characters you want to allow in the form control.

A11: To control the range of values users can enter into number-based form controls, you can use the min and max attributes. These attributes allow you to set the lower and upper bounds of the value entered into the form control.

A12: To add a pattern validation to a form control, you can use the pattern attribute with a regular expression as the value. This ensures that the data entered by the user matches a particular pattern.

A13: Real-world applications of pattern validation include checking if a credit card number or a zipcode is in the correct format.

A14: You can style form controls that have passed or failed validations using the :valid and :invalid pseudo-classes in CSS.

A15: Some limitations of built-in validations include not being able to validate that a password input and password confirmation input have the same value, or validating that a username has not already been taken. They also limit what can be done with styling the validation messages and the content within them.

A16: It's important to also have server-side validations to ensure the integrity of any user data coming into our systems. Client-side validations are not a silver bullet for ensuring users enter the correct data.
